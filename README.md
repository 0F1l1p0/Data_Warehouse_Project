# Data_Warehouse_Project
Building a data warehouse using MS SQL Server through ETL processes and different types of data modeling. This specific project is done through the medallion architecture data design pattern

##__Project Tasks__

Develeoping a data warehouse using MS SQL Server to enable analyyitcal reporting and real life decision-making based off the provided data in the data warehouse



##__Data Sourching__

Importing data from two different systems (In this case that would be CRM and ERP) that are formated as CSV files.



##__Data Quality Management__

Applying data cleaning skills that would involve detecting and correcting innacurate or inconsitently formatted data to ensure the maximum possible accuracy for business analytics.
Inconsistencies such as handling missing values, detecting outliers within the dataset, removing duplicates and value standardization throughout the entire data warehouse.



##__Data Integration__

Combining both of the given sources into a single data model that would later be used for analytical queries, while additionally providing clear documentation of the data model
to keep transparency for stakeholders and analytics teams



##__Data Analytics__

Handle SQL-based analytics in order to delvier detailed insights into
  -Customer Behaviour
  -Sales Trends
  -Product Performance
